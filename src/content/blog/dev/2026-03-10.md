prompt:
```
你現在是一個精通 Astro 和 Tailwind CSS 的資深前端工程師。
請幫我從零建立一個名為 "dev-diary" 的靜態網站專案，作為我的每日開發紀錄。

【核心目標】
這是一個「極簡 (Deadly Simple)」的 Markdown 網誌，以乾淨的文字閱讀體驗為主，不需要任何花俏的動畫或複雜的元件。我的目的是每天只花 10 分鐘速記開發心得，並推播到 GitHub 累積 commit streak。

【技術棧與配置】
- 框架：Astro (請設定為靜態生成 SSG 模式)
- 樣式：Tailwind CSS (請使用 @astrojs/tailwind 官方整合)
- 內容管理：Astro Content Collections (請配置 src/content/blog 來存放 .md 檔案)
- 排版套件：請安裝並設定 @tailwindcss/typography，用 `prose` class 來渲染 Markdown。

【功能與版面需求】
1. 基礎版面 (Layout)：
   - 包含一個極簡的 Header（顯示網站名稱）。
   - 主體內容區塊需要置中，最大寬度設定為 `max-w-3xl`，確保閱讀舒適度。
   - 預設提供友善的深色模式 (Dark mode) 樣式。
2. 首頁 (index.astro)：
   - 條列出所有的開發日誌，以日期反序排列（最新的在最上面），列表只需顯示「日期」與「標題」。
3. 自動化腳本 (new_post.sh)：
   - 在專案根目錄建立一個 `new_post.sh`。
   - 執行時，自動在 src/content/blog/ 建立一個以「YYYY-MM-DD.md」命名的檔案。
   - 檔案內需自動帶入基本的 Frontmatter (title, pubDate, description)。
4. CI/CD 部署 (GitHub Actions)：
   - 建立 `.github/workflows/deploy.yml`。
   - 設定當推播至 `main` 分支時，自動 build 專案並透過 GitHub Pages 發布 (請使用官方推薦的 Astro 部署 workflow)。

【請立刻執行】
1. 初始化專案結構、安裝必要的 dependencies 並設定好設定檔。
2. 建立基礎的 Layout 與首頁。
3. 幫我建立一篇測試用的 Markdown 文章（例如標題為："[Daily] 測試在 Arch Linux 下的開發環境設定與 AI Agent 框架研究"），讓我確認 Markdown 渲染與 Tailwind Typography 的排版效果。
4. 提供完成後的專案結構說明，以及告訴我接下來要如何在 GitHub 上開啟 Pages 功能。
```
